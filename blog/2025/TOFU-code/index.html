In TOFU, We have 2 stage for evaluating LLM unlearning performances.

First, we finetune the model with all of fictitious authors (200 people). Second, we unlearn the model with split of them.

Tokenizer

...
tokenizer = AutoTokenizer.from_pretrained(model_id)
tokenizer.pad_token = tokenizer.eos_token
...

Why tokenizer.pad_token to tokenizer.eos_token ? Most LLMs like GPT are trained without a dedicated pad token. They natually process text as continuous streams using the end-of-sequence (EOS) token to mark text endings. During training, we often need to batch multiple sequences together. These sequences might have different lengths. To make them uniform length for batch processing, we need padding. Since the model is already trained to understand EOS tokens. Using EOS as padding is more natural than introducing a new token the model hasn’t seen during pre-training.

Here, the locuslab/TOFU repository provides the TOFU benchmark datasets. The cfg.split parameter allows you to choose the split between the “forget” and “retain” datasets. The full option is selected for fine-tuning on all items associated with fictitious authors.
